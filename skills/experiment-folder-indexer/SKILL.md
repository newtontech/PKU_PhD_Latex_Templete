# Experiment Folder Indexer Skill

Use this skill as the first step in any evidence-bound revision workflow. It scans experiment directories, identifies evidence files (notebooks, scripts, logs, figures, data files), and assigns stable IDs to each one. This ledger becomes the foundation for claim-evidence mapping, results consistency checks, and submission QA.

## When to Use

- Before running claim-evidence-map to build the evidence surface the matrix will reference.
- Before starting a manuscript revision that requires tracing results back to experiments.
- When onboarding a new team member who needs to understand what evidence exists.
- After completing a batch of experiments, to register new results into the evidence ledger.
- When the project has grown and the existing inventory may be stale.

## Input

- **project-root-or-experiments-dir**: Path to the project root or a specific experiments directory.
  - If a project root is given, the skill searches standard subdirectories.
  - If a specific directory is given, only that tree is scanned.

## Workflow

1. **Identify scan targets.** From the given root, locate the following directories (skipping any that do not exist):
   - `experiments/`
   - `notebooks/`
   - `results/`
   - `logs/`
   - `figures/`
   - `tables/`
   - `data/`
   - `scripts/`
   - `output/`
   - Any custom paths provided by the user.

2. **Scan for evidence files.** Recursively walk each target directory. Identify files by extension:

   | Extension | Type | Notes |
   | --- | --- | --- |
   | `.ipynb` | notebook | Jupyter notebook; may contain code, outputs, figures |
   | `.py`, `.R`, `.m`, `.jl` | code | Analysis or experiment scripts |
   | `.log`, `.out` | log | Execution logs, stdout captures |
   | `.csv`, `.tsv` | data | Tabular results data |
   | `.json`, `.jsonl` | data | Structured results or metrics |
   | `.xlsx`, `.xls` | data | Spreadsheet results |
   | `.pdf` | figure | Vector figures or result summaries |
   | `.png`, `.jpg`, `.jpeg`, `.svg`, `.eps` | figure | Raster or vector figures |
   | `.tex` | table | LaTeX table source |
   | `.md` | document | Notes, README, supplementary text |
   | `.pkl`, `.npy`, `.mat`, `.h5`, `.hdf5` | data | Binary data files |
   | `.bib` | reference | Bibliography files |

   Skip hidden files (dot-prefix), `.git/`, `__pycache__/`, `node_modules/`, and `*.aux` LaTeX intermediates.

3. **Assign stable IDs.** Assign a sequential ID to each discovered file:
   - Format: `E001`, `E002`, `E003`, ...
   - IDs are assigned in lexicographic order of the file path to ensure determinism across runs.
   - If an existing `evidence_ledger.jsonl` is found, reuse IDs for unchanged paths and assign new IDs for new files.

4. **Determine evidence status.** For each file, assign a status:

   | Status | Meaning |
   | --- | --- |
   | `READY` | File exists, is readable, and likely usable as evidence |
   | `NEEDS_PARSING` | Requires conversion or execution (e.g., `.ipynb` needs output extraction) |
   | `MISSING_DEPENDENCY` | File references another file that does not exist (e.g., script imports missing module) |
   | `UNKNOWN` | Role or content cannot be determined automatically |

   Status heuristics:
   - `.ipynb` without outputs: `NEEDS_PARSING`.
   - `.py` scripts that import local modules not found: `MISSING_DEPENDENCY`.
   - All readable figure and data files: `READY`.
   - Files with unrecognized extensions: `UNKNOWN`.

5. **Generate evidence ledger.** Write `evidence_ledger.jsonl` with one JSON object per line:

   ```json
   {
     "id": "E001",
     "path": "experiments/exp01/run.ipynb",
     "type": "notebook",
     "status": "NEEDS_PARSING",
     "description": "Jupyter notebook for experiment 01 baseline run",
     "size_bytes": 45230,
     "last_modified": "2025-11-15T14:30:00Z",
     "provenance": "",
     "tags": ["baseline", "training"]
   }
   ```

6. **Generate figure and table inventory.** Write `figure_table_inventory.md` listing all figures and tables with their provenance:

   ```markdown
   # Figure and Table Inventory

   ## Figures
   | ID | Path | Format | Provenance | Used In |
   | --- | --- | --- | --- | --- |
   | E003 | figures/architecture.pdf | PDF | Created manually | Section 3.1 |
   | E007 | results/loss_curve.png | PNG | Generated by train.py | Section 4.2 |

   ## Tables
   | ID | Path | Format | Provenance | Used In |
   | --- | --- | --- | --- | --- |
   | E012 | results/metrics.csv | CSV | Output of evaluate.py | Table 2 |
   | E015 | tables/comparison.tex | TEX | Written manually | Table 3 |

   ## Unlinked Files
   (Figures and tables not referenced in any known manuscript file)
   ```

## Output

- **evidence_ledger.jsonl**: One JSON object per line for every discovered evidence file, with stable IDs, paths, types, and statuses.
- **figure_table_inventory.md**: Markdown inventory of all figures and tables with provenance information.
- **indexer_summary.txt**: Brief summary of the scan.

  ```
  === Experiment Folder Indexer Summary ===
  Root: /path/to/project
  Directories scanned: 6
  Total files indexed: 47
  Status breakdown:
    READY: 32
    NEEDS_PARSING: 8
    MISSING_DEPENDENCY: 3
    UNKNOWN: 4
  Figures: 12
  Tables: 5
  Notebooks: 4
  Scripts: 8
  Data files: 11
  Logs: 7
  ```

## Rules

- This skill is read-only: it must not execute notebooks, run scripts, or modify any files.
- Skip directories and files matching common ignore patterns (`.git`, `__pycache__`, `node_modules`, `.DS_Store`, LaTeX intermediates).
- Assign IDs deterministically (lexicographic path order) so reruns produce stable IDs.
- If an existing `evidence_ledger.jsonl` is present, merge rather than overwrite: keep existing entries for unchanged paths, update modified entries, add new entries, and mark removed entries.
- For `.ipynb` files, inspect the notebook JSON to check whether outputs are present (affects NEEDS_PARSING status).
- File descriptions are generated from path heuristics and filenames; they should not fabricate content descriptions that are not evident from the path or filename.
- Maximum scan depth: 5 directory levels to avoid scanning unrelated nested structures.
