# LaTeX Compilation Fix Skill

Automatically diagnose and repair the most common LaTeX compilation errors encountered with the PKU PhD thesis template (ctexbook, XeLaTeX). Reduce manual log-trawling by classifying errors and applying safe, targeted fixes.

## When to Use

- After `xelatex COEmain.tex` exits with errors or critical warnings.
- When the build log shows repeated warnings that would fail pre-review (overfull hboxes, undefined citations).
- Before submission to ensure a clean compilation with zero errors.
- After adding new packages, figures, or bibliography entries that may conflict with the PKU template.

## Input

- **tex-main-file**: Path to the main `.tex` file (default `COEmain.tex`).
- The skill reads the `.log` file generated by the most recent compilation attempt.

## Workflow

1. **Run first XeLaTeX pass.**
   ```bash
   xelatex -interaction=nonstopmode -file-line-error COEmain.tex
   ```
   Capture the exit code and log.

2. **Parse and classify errors.** Extract error lines from `.log` and match against known patterns:

   | Pattern | Cause | Fix Strategy |
   | --- | --- | --- |
   | `Undefined control sequence` | Misspelled command or missing package | Check typo; add `\usepackage` if package missing |
   | `Missing $ inserted` | Math content outside math mode | Wrap offending text in `$...$` or `\(...\)` |
   | `Missing } inserted` | Unbalanced braces | Locate unmatched `{` at reported line; add/remove brace |
   | `Citation 'X' undefined` | Missing `.bib` entry or typo | Run `gbt7714-reference-check` skill; add entry or fix key |
   | `File 'X' not found` | Missing figure or input file | Verify path, extension, and case sensitivity |
   | `Font shape 'X' not available` | Missing font or wrong font name | Check system CJK fonts; use `fc-list` to verify |
   | `Package X not found` | Missing LaTeX package | Install via `tlmgr install X` or distro package manager |
   | `Too many unprocessed floats` | Figure/table overflow | Insert `\clearpage` or relax placement to `[htbp]` |
   | `Overfull \hbox` | Line exceeds text width | Rewrite sentence, add hyphenation hint, or adjust margins |
   | `Underfull \vbox` | Poor page break | Adjust text, add `\vfill`, or accept if minor |

3. **Apply fixes iteratively.** For each classified error:
   - Locate the source `.tex` file and line number.
   - Apply the minimal safe fix.
   - Re-run `xelatex`.
   - Stop after 5 iterations to avoid infinite loops.

4. **Run full build pipeline.** Once errors are resolved, run the complete sequence:
   ```bash
   xelatex COEmain.tex
   bibtex COEmain
   xelatex COEmain.tex
   xelatex COEmain.tex
   ```

5. **Generate fix report.** Write `compilation_fix_log.md` documenting every change made.

## Output

### `compilation_fix_log.md`

```markdown
# LaTeX Compilation Fix Log

## Initial State
- Main file: COEmain.tex
- Errors: 3
- Warnings: 12

## Fixes Applied

### Iteration 1
- **Error**: `Missing $ inserted` (body/COEchap2.tex:145)
- **Fix**: Wrapped `E = mc^2` in math mode → `$E = mc^2$`
- **Result**: PASS

### Iteration 2
- **Error**: `Package siunitx not found`
- **Fix**: Installed via `tlmgr install siunitx`
- **Result**: PASS

## Final State
- Errors: 0
- Warnings: 2 (acceptable overfull hboxes in URLs)
- PDF: COEmain.pdf (142 pages)
```

## Rules

- Always prefer the minimal fix that resolves the error without changing content meaning.
- Never delete content to silence an error unless the user explicitly approves.
- If the same error persists after 3 iterations, stop and report the log excerpt to the user.
- Preserve intermediate files (`.aux`, `.bbl`, `.log`, `.toc`) for debugging.
- Use `tlmgr` for TeX Live or the OS package manager for MiKTeX/system installs.
- For font issues on Linux, verify with `fc-list :lang=zh` that Songti/Heiti fonts are present.
